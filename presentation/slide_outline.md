# Slide 2 — Motivation

## Why this project matters

Large language models often produce fluent and semantically similar text.

However, even when models answer the same prompt, they may differ in:

- sentence structure;
- punctuation habits;
- word choice;
- readability;
- function-word usage;
- lexical diversity.

## Core idea

Semantic similarity tells us what the text is about.  
Stylometry tells us how the text is written.

**Speaker notes:**  
Most LLM evaluation focuses on meaning, correctness, or human versus AI detection. My project asks a more specific question: even if different models produce semantically similar answers, do they write in measurably different styles? This is why I use stylometry. Instead of only looking at content, I measure structural and linguistic writing patterns.

# Slide 3 — Research Question and Objectives

## Research question

Can stylometric features distinguish texts generated by different LLM families?

## Objectives

1. Build a balanced corpus of LLM-generated texts.
2. Extract measurable stylometric features.
3. Test whether these features differ across model families.
4. Compare semantic clustering with stylometric evidence.
5. Train classifiers to predict model family.
6. Critically discuss the effectiveness and limitations of the approach.

**Speaker notes:**  
The project has both descriptive and predictive goals. First, I want to know whether stylistic features differ statistically across model families. Second, I want to test whether those features are strong enough to predict which model generated a text. Importantly, the goal is not simply to get high accuracy, but to understand whether the methodology is effective.

# Slide 4 — Dataset and Experimental Design

## Dataset

| Item | Value |
|---|---:|
| Generated texts | 1,000 |
| Model families | 5 |
| Texts per model | 200 |
| Genres | 4 |
| Texts per genre | 250 |
| Texts per model per genre | 50 |
| Stylometric features | 43 |
| Prompt groups | 200 |

## Genres

Narrative, descriptive, argumentative, dialogue

**Speaker notes:**  
The dataset is balanced. Each model generated 200 texts, and each genre has 250 texts. This design is important because it avoids bias from unequal class sizes or genre imbalance. Each model answered the same controlled prompt set, making the comparison fairer.

# Slide 5 — Methodology Overview

## Pipeline

1. Controlled prompt design
2. API generation from five LLM families
3. Cleaning and length validation
4. Stylometric feature extraction
5. Semantic mapping with sentence embeddings
6. Statistical testing
7. Classification modeling
8. Streamlit dashboard for interactive exploration

## Formal task

Given a generated text \(x_i\), extract stylometric features:

\[
\phi(x_i) \in \mathbb{R}^{43}
\]

Then predict:

\[
f: \mathbb{R}^{43} \rightarrow \{\text{GPT, Claude, DeepSeek, Gemini, Mistral}\}
\]

**Speaker notes:**  
The project is structured as a reproducible pipeline. Each generated text is transformed into a vector of 43 stylometric features. The classification task is to predict the model family from these features. I also compare this with sentence-embedding maps to see whether model identity appears in semantic space.

# Slide 6 — Stylometric Features

## Feature groups

1. **Length and sentence structure**
   - word count;
   - sentence count;
   - average sentence length;
   - long-sentence ratio.

2. **Punctuation**
   - comma count;
   - period count;
   - dash count;
   - punctuation per 100 words.

3. **Lexical diversity**
   - type-token ratio;
   - Yule’s K;
   - Honore’s R;
   - MATTR.

4. **Markers and function words**
   - function-word ratio;
   - transition markers;
   - hedging markers;
   - AI-style markers.

5. **Readability**
   - Flesch reading ease;
   - Gunning Fog index.

**Speaker notes:**  
The features are interpretable. They describe how a text is written rather than what topic it discusses. This includes sentence length, punctuation style, lexical diversity, readability, and discourse markers. These features make it possible to compare model writing styles quantitatively.
